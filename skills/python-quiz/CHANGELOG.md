# Changelog — python-quiz

## [Unreleased]

### Changed
- Добавлен `allowed-tools` в frontmatter: Read, Write, Bash, AskUserQuestion — ограничивает доступные инструменты минимально необходимым набором

## [1.9.1] - 2026-07-13

### Changed
- Заменён нестандартный блок `parameters` на `argument-hint: "[init]"` — корректный механизм подсказки аргументов в Claude Code

## [1.9.0] - 2026-07-13

### Added
- `parameters` schema в YAML frontmatter: поле `command` с `enum: ["init"]` — обеспечивает автодополнение и плейсхолдер в Claude Code и совместимых AI-клиентах

## [1.8.0] - 2026-07-13

### Fixed
- Init 2: batch-генерация вопросов по 5–8 тем за проход вместо посекционного чтения — значительно ускоряет init-режим
- Quiz 2.2: явный запрет показывать поле `answer` при задавании вопроса — ответы хранятся только для Фазы 3
- Quiz 2.2: явный запрет запускать bash для чтения вопросов в процессе теста — устраняет tool trace в opencode
- Quiz 2.2: `multiSelect` теперь явно `true` для типа "multi" и `false` для "single" — устраняет потерю multi-select

## [1.7.0] - 2026-07-12

### Changed
- Quiz 1: лимит разнообразия снижен до ≤1 вопроса на subtopic (было только ≤2 на topic) — устраняет кластеризацию по подтемам
- Quiz 1: алгоритм выборки заменён на round-robin по темам — чередование гарантировано
- Quiz 1: запрет двух подряд вопросов из одной темы при финальной расстановке по сложности

## [1.6.0] - 2026-07-12

### Fixed
- Quiz 2.2: Claude больше не отвечает за пользователя — после short answer и coding task явный запрет на продолжение генерации до получения ответа
- Init 2: вопросы обязаны быть самодостаточными (не ссылаться на внешний контекст, не обрываться)
- Init 2.5 (новый шаг): проверка качества банка перед сохранением — удаление/регенерация некорректных вопросов

## [1.5.0] - 2026-07-12

### Added
- Режим `python-quiz init`: строит банк вопросов через Hierarchical TOC (.md заголовки ##/###)
- Чтение .md файлов посекционно — не весь файл сразу, снижает пиковую нагрузку токенов
- Банк вопросов `python-questions.json`: 150–300 вопросов, структура с topic/subtopic/difficulty
- Параметр `questions_per_topic` (по умолчанию 6, диапазон 5–7) в CONFIG.md

### Changed
- Режим quiz теперь семплирует из готового банка (не читает .md при каждом запуске)
- Семплирование: баланс типов + не более 2 вопросов на тему + нарастание сложности
- CONFIG.md: добавлены `questions_per_topic` и `bank_file`

## [1.4.0] - 2026-07-12

### Changed
- Фаза 1: генерируются только вопросы — без правильных ответов и объяснений
- Таймер: только старт и стоп, без промежуточных проверок после каждого ответа
- Фаза 3: батчевая оценка всех ответов одним проходом после завершения теста
- ERROR-REVIEW.md: объяснения генерируются здесь по требованию, не заранее

## [1.3.0] - 2026-07-12

### Changed
- Разбор ошибок вынесен в `references/ERROR-REVIEW.md` — загружается только по запросу пользователя

## [1.2.0] - 2026-07-12

### Changed
- Таймер: убран внешний `scripts/timer.sh`, заменён на inline bash one-liners прямо в SKILL.md — нет зависимости от пути к скриллу
- Проверка времени теперь после **каждого** ответа (вместо каждые 5 вопросов)
- DRY: таблица распределения типов вопросов убрана из SKILL.md, единственный источник — `references/CONFIG.md`

## [1.1.0] - 2026-07-12

### Added
- `scripts/timer.sh`: bash-таймер (start / check / stop через epoch-файл в /tmp)
- `max_time_minutes` в CONFIG.md (по умолчанию 23 мин); превышение → тест не засчитан
- Pre-generation фаза: все вопросы генерируются до старта таймера, без задержек во время теста
- Проверка времени каждые 5 вопросов и в финале
- Предупреждение при остатке < 3 мин
- Fallback для окружений без Bash tool

### Changed
- Переименован с `python-interview` в `python-quiz`
- Алгоритм разбит на 3 явные фазы: Подготовка → Тест → Финал

## [1.0.0] - 2026-07-12

### Added
- Initial release: интерактивный тест Python по материалам .md файлов в директории
- Поддержка 4 типов вопросов: single-choice, multi-choice, short answer, coding task
- Итоговый отчёт: % правильных ответов, оценка по шкале
- Разбор ошибок по запросу с объяснениями
- Настройки теста через `references/CONFIG.md`
