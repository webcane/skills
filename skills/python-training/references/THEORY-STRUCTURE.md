# Структура теоретического разбора

При показе теории используй markdown-форматирование со следующими секциями (если они применимы к теме):

1. **Определение и мотивация** — кратко о сути и почему это важно (1–2 абзаца)
2. **Основные концепции** — ключевые понятия с минимальными примерами кода
3. **Детальный разбор** — как это работает под капотом. Для каждой важной идеи — отдельный подраздел с кодом и пояснением
4. **Примеры** — 2–4 практических примера разной сложности
5. **Сравнение альтернатив** — таблица, если уместно (например, list vs generator, mutable vs immutable)
6. **Когда использовать / когда не использовать** — чёткие критерии выбора
7. **Практический пример** — один сквозной пример, который сводит всё вместе

Ориентируйся на формат: мотивация → код → разбор → сравнение → практика. Не сохраняй в файл — весь материал показывай в сообщении.
